## How to use this skill

The user's argument (`$ARGUMENTS`) is the documentation subject.

### Steps

1. **Derive the filename** from `$ARGUMENTS`: lowercase, spaces → hyphens, strip special characters, append `.md`.
   - `"API reference"` → `api-reference.md`
   - `"architecture overview of the provider system"` → `architecture-overview-of-the-provider-system.md`

2. **Check for an existing file** at `docs/<filename>.md`. If it exists, update it in place rather than creating a duplicate.

3. **Read source files** if the subject requires code knowledge. Project source lives under `promptlab/`:
   - Routers: `promptlab/app/routers/`
   - Schemas: `promptlab/app/schemas.py`
   - Data access: `promptlab/app/crud.py`
   - LLM service: `promptlab/app/llm_service.py`
   - Models: `promptlab/app/models.py`

4. **Write** the finished document to `docs/<filename>.md`.

### Output rules

- Use clear headings, tables for tabular data, fenced code blocks for examples.
- Never create the file at the project root or inside `promptlab/` source directories.
- Do not add a "generated by" footer.
